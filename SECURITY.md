# Security policy

## Supported versions

Security fixes are applied to the latest version on the `main` branch.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability or include private
project data, database contents, tokens, or credentials in a report.

Use GitHub's private vulnerability reporting feature from the repository's
Security tab. Include reproduction steps, affected versions, impact, and any
suggested mitigation. You should receive an acknowledgement through the private
report within seven days.

If private vulnerability reporting is not available, contact the repository
owner privately before sharing technical details.
